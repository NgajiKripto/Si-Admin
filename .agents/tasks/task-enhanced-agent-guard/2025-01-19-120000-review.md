# Action permissions, knowledge audit, and zero-width detection for Agent Guard (v3)

The agent-guard system adds read-only mode with granular action permissions, a knowledge-content auditor blocking prompt injection in knowledge entries, and zero-width Unicode detection in the input sanitizer. Both blocking issues from v2 have been resolved: `ZERO_WIDTH_CHARS_PATTERN` is now exported without the `/g` flag (a fresh `/g` regex is constructed at the `.replace()` call site via `new RegExp(pattern.source, "g")`), and the `system:` pattern is anchored to line-start with `/^\s*system\s*:/im`.

**Watch for:** No blocking concerns remain. The `isActionAllowed` enforcement is demonstrated in `example-usage.ts` but not wired into the actual request pipeline — this is acknowledged scaffolding, not a bug in this PR's scope.

**Verdict**: APPROVED

## High-level view

The stateful-regex bug is resolved. `ZERO_WIDTH_CHARS_PATTERN` stores the detection pattern without `/g`, so `.test()` calls never drift `lastIndex`. The sanitizer constructs a one-shot global regex from `.source` only at the `.replace()` call site. Both consumers are now stateless across invocations.

The `system:` false-positive surface is substantially reduced. `/^\s*system\s*:/im` only matches at line-start, so inline occurrences ("Operating system: Windows") no longer trigger rejection while the actual injection vector (a line beginning a system prompt block) is still caught.

## Zero-width regex fix verification

The exported constant in `input-sanitizer.ts`:

```typescript
export const ZERO_WIDTH_CHARS_PATTERN =
  /[\u200B\u200C\u200D\uFEFF\u202A-\u202E\u2060\u2066-\u2069]/;
```

No `/g` flag. The sanitizer's `.replace()` constructs a fresh regex per call:

```typescript
sanitizedInput = sanitizedInput.replace(
  new RegExp(ZERO_WIDTH_CHARS_PATTERN.source, "g"),
  ""
);
```

This is a local, ephemeral regex that cannot leak state between invocations. The knowledge auditor calls `.test()` on the same flag-free constant, so `lastIndex` is always 0.

## `system:` pattern anchoring

```typescript
{ pattern: /^\s*system\s*:/im, label: "System prompt fragment: 'system:'" }
```

The `^` anchor with the `m` (multiline) flag restricts matches to line-start. The `i` flag preserves case-insensitivity. Mid-sentence uses of "system:" are no longer rejected.
